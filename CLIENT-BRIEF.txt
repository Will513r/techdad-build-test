HOW WE BUILD  (Tech Dad Media house style — keep this UNIFORM across sites)
=======================================
The look is designed once as a single self-contained index.html — that is our
style and source of truth. Then we convert it to modular PHP:

  1. Pull every asset out of the HTML demo:
       images -> /assets/images   (WebP)
       CSS    -> /assets/css      (critical.css + style.css)
       JS     -> /assets/js
  2. Repeated markup becomes includes (head, header, footer, CTA strips).
  3. Content becomes PHP data arrays — never hardcode it into the markup.
  4. Each service renders from ONE template, driven by /data/services.php.

Standard structure (mirrors github.com/Will513r/Shawns-Landscape — smaller
sites use the SAME skeleton with fewer pages):

  /index.php                    homepage
  /about.php /contact.php /reviews.php /areas-we-serve.php /404.php
  /.htaccess                    clean URLs (/services/roof-soft-washing, /contact)
  /includes/                    shared partials, included on every page
      config.php                site constants (SITE_URL, phone, REVIEW_RATING…)
      head.php                  <head>: title/meta/canonical/OG, built from $meta
      header.php   footer.php
      schema-helpers.php        JSON-LD builders (LocalBusiness, FAQ, Service)
      contact-strip.php  mobile-sticky-cta.php  nearby-areas.php  (reusable blocks)
  /data/                        content as PHP arrays (DRY)
      services.php              keyed by slug: name, full_name, description,
                                symptoms[], anti_complaint[], cta_suffix
      cities.php   reviews.php   projects.php
  /services/
      index.php                 services hub
      service-template.php      renders ANY service from data/services.php
                                (?service=slug, optional ?city=slug for local pages)
  /assets/css   /assets/js   /assets/images(webp)
  /api/                         form / lead endpoints (e.g. submit-lead.php)

Page pattern: each page sets $meta (title/description/canonical), then
include head.php -> header.php -> content -> footer.php. Keep content in /data,
markup DRY, and SEO (schema + meta) baked in from the first commit.
